# 科学家与大国工匠精神展示平台 - 使用说明

## 一、平台概述

这是一个专为科技教育与文化传播设计的现代化展示平台，旨在宣传科学家和大国工匠的感人事迹，弘扬时代精神。平台适用于高校、展馆、公共服务机构等场所。

## 二、功能特性

### 1. 主页展示
- **英雄横幅**：大型视觉横幅展示平台主题和统计数据
- **特色人物**：首页展示3位重点人物，配有大图和简介
- **分类展示**：科学家和工匠分别展示，便于分类浏览
- **精神传承**：展示科学家与工匠的核心精神品质

### 2. 智能搜索功能
- **搜索按钮**：点击右上角🔍按钮打开搜索侧边栏
- **实时建议**：输入姓名或关键词时，自动显示匹配的人物建议
- **高亮匹配**：建议列表中高亮显示匹配的文字
- **快速选择**：点击建议直接跳转到该人物

### 3. 高级筛选
- **类别筛选**：全部/科学家/工匠，点击即时生效
- **领域筛选**：按专业领域筛选（物理学、航天工程、医学等）
- **年代筛选**：按时代分类（近代、现代、当代）
- **组合筛选**：多个筛选条件可以组合使用
- **一键清空**：快速清除所有筛选条件

### 4. 人物详情页
点击任意人物卡片，打开详细信息模态框，包含：

- **基本信息**：姓名、类别、领域、生卒年份
- **操作按钮**：
  - ❤️ 收藏按钮：收藏喜欢的人物（本地存储）
  - 🔗 分享按钮：分享人物事迹给他人
- **人物简介**：完整的生平介绍
- **视频播放**：观看相关纪录片（本地视频）
- **音频讲解**：收听语音讲解（如有）
- **人生历程**：时间线形式展示关键节点
- **主要成就**：列举重要贡献和成果
- **精神品质**：展示人物的精神特质
- **相关标签**：关键词标签

### 5. 收藏系统
- **本地存储**：收藏数据保存在浏览器本地，无需登录
- **持久保存**：清除缓存前数据永久保留
- **状态显示**：已收藏的人物显示❤️红心图标
- **快速切换**：再次点击取消收藏

### 6. 分享功能
- **智能分享**：
  - 移动设备：使用系统原生分享功能
  - 桌面设备：自动复制链接到剪贴板
- **直接链接**：生成带参数的URL，可直接打开指定人物详情
- **提示反馈**：操作成功后显示Toast提示

### 7. 浏览历史
- **自动记录**：每次查看人物详情自动记录
- **最近20条**：保留最近浏览的20位人物
- **本地存储**：数据保存在浏览器中

### 8. 访问统计
- **浏览次数**：统计每位人物的查看次数
- **热门排行**：可按浏览量排序（功能预留）

## 三、快捷操作

### 键盘快捷键
- **ESC键**：关闭模态框/关闭搜索侧边栏/隐藏建议

### 鼠标操作
- **点击卡片**：查看人物详情
- **点击蒙层**：关闭模态框
- **点击建议**：快速搜索人物

## 五、如何添加新人物

1. **打开 data.js 文件**
2. **在 peopleData 数组末尾添加新对象**，参考以下模板：

```javascript
{
    id: 9,  // 新的ID，递增
    name: "人物姓名",
    category: "scientist",  // 或 "craftsman"
    field: "专业领域",
    era: "当代",  // 或"近代"、"现代"
    birth: "1950",
    death: "",  // 如健在则留空
    image: "images/人物姓名.jpg",
    video: "https://www.bilibili.com/video/BV号",  // B站视频链接
    audio: "",  // 音频文件路径
    shortDesc: "简短描述，一两句话介绍主要贡献。",
    fullDesc: "完整的人物介绍，包括生平、贡献、影响等。",
    timeline: [
        {
            year: "1950",
            title: "出生",
            desc: "出生地点"
        },
        // ... 更多时间节点
    ],
    achievements: [
        {
            title: "成就标题",
            desc: "成就详细描述"
        },
        // ... 更多成就
    ],
    spirit: [
        {
            icon: "🔬",
            name: "精神名称",
            desc: "精神描述"
        },
        // ... 更多精神特质
    ],
    tags: ["标签1", "标签2", "标签3"]
}
```

3. **准备人物图片**：
   - 图片尺寸建议：800x1000像素以上
   - 格式：JPG或PNG
   - 文件名：与 name 字段一致
   - 存放位置：images/ 文件夹

4. **准备人物视频**（可选）：
   - 视频格式：MP4 (H.264编码)
   - 文件名：与 name 字段一致（如：钱学森.mp4）
   - 存放位置：videos/ 文件夹
   - 详细说明：参考 videos/视频文件说明.md

5. **保存并刷新**页面即可看到新人物

## 七、注意事项

1. **图片文件**：请确保所有人物图片已放入 images/ 文件夹
2. **视频链接**：B站视频链接需要有效且公开
3. **本地存储**：收藏和历史记录保存在浏览器中，清除浏览器数据会丢失
4. **文件编码**：所有文件使用 UTF-8 编码
5. **跨平台**：文件可以在 Windows、Mac、Linux 上直接打开

## 八、常见问题

### Q1: 为什么看不到图片？
**A:** 请确认：
- images/ 文件夹存在
- 图片文件名与 data.js 中的 image 字段完全一致（包括扩展名）
- 图片文件已正确下载

### Q2: 视频无法播放？
**A:** 请检查：
- videos/ 文件夹存在
- 视频文件名与 data.js 中的 video 字段完全一致
- 视频格式为MP4 (H.264编码)
- 文件路径正确：videos/人物姓名.mp4
- 详细排查：参考 videos/视频文件说明.md

### Q3: 收藏功能失效？
**A:** 可能原因：
- 浏览器禁用了本地存储
- 使用了无痕模式
- 浏览器清除了缓存数据

### Q4: 如何部署到服务器？
**A:**
1. 将整个文件夹上传到Web服务器
2. 配置Web服务器指向 index.html
3. 确保所有文件路径正确

### Q5: 如何备份收藏和历史？
**A:** 目前数据存储在浏览器中，暂不支持导出。未来版本将添加账户系统和云端同步功能。
