## 💡 两种实现方案

### 方案A：纯前端版（当前）

**优点**：
- ✅ 部署简单，无需服务器
- ✅ 开发快速，维护方便
- ✅ 适合展示和演示

**限制**：
- ❌ 无法实现真实用户系统
- ❌ 数据不能持久化到服务器
- ❌ 功能相对有限

**适用场景**：
- 校园展示屏
- 静态网站展示
- 原型演示

### 方案B：前后端分离版（推荐）

**优点**：
- ✅ 功能完整
- ✅ 数据持久化
- ✅ 支持多用户
- ✅ 可扩展性强

**技术架构**：
```
前端：Vue.js / React
后端：Node.js + Express
数据库：MySQL
存储：阿里云OSS
部署：nginx + PM2
```

**适用场景**：
- 正式上线运营
- 多用户使用
- 需要数据统计

## 🔐 用户系统设计

### 数据库表结构

#### users 用户表
```sql
CREATE TABLE users (
  id INT PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50) UNIQUE,
  password VARCHAR(255),
  email VARCHAR(100),
  avatar VARCHAR(255),
  create_time TIMESTAMP,
  last_login TIMESTAMP
);
```

#### favorites 收藏表
```sql
CREATE TABLE favorites (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT,
  person_id INT,
  create_time TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### API接口设计

#### 用户相关
```
POST   /api/register        注册
POST   /api/login           登录
GET    /api/profile         获取个人信息
PUT    /api/profile         更新个人信息
POST   /api/logout          登出
```

#### 收藏相关
```
GET    /api/favorites       获取收藏列表
POST   /api/favorites       添加收藏
DELETE /api/favorites/:id   取消收藏
```

#### 内容相关
```
GET    /api/people          获取人物列表
GET    /api/people/:id      获取人物详情
POST   /api/people          创建人物（管理员）
PUT    /api/people/:id      更新人物（管理员）
DELETE /api/people/:id      删除人物（管理员）
```
